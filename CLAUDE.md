# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project purpose

Noise-Detection Glasses — a wearable device that classifies ambient sounds via YAMNet and signals the wearer through NeoPixel LEDs on a glasses frame. The current codebase is a laptop-side prototyping and validation suite; the ESP32-S3 firmware is the end target.

## Common commands

All scripts are run from the `YamnetTest/` directory.

```bash
# Live mic demo (no hardware needed)
python yamnet_mic_test.py

# Generate TFLite models + ESP32-S3 feasibility report (run once)
python yamnet_esp32_feasibility.py

# Serial bridge to XIAO ESP32-S3
python yamnet_serial.py --list-ports
python yamnet_serial.py --port /dev/cu.usbmodem1101 [--threshold 0.6]
python yamnet_serial.py --port dummy --dry-run       # no hardware

# Accuracy tests
python yamnet_accuracy_test.py --esc50-dir downloads/ESC-50-master
python yamnet_urbansound_test.py [--limit 200] [--no-tflite]
```

## Architecture

The pipeline has two distinct modes:

**Laptop-only (prototyping):** `yamnet_mic_test.py` captures mic audio into a rolling ring buffer, runs YAMNet (loaded from TF Hub), and prints top-5 predictions with LED color suggestions every 0.5 s.

**Laptop → Hardware bridge:** `yamnet_serial.py` does the same inference on the laptop, then serializes results over USB serial to the XIAO ESP32-S3. The Arduino sketch on the ESP32 side receives those commands and drives the NeoPixels. This avoids porting TFLite Micro until needed.

**Serial protocol** (defined in `yamnet_serial.py`):
- `C:<category>:<confidence>:<brightness>` — new classification + brightness
- `B:<brightness>` — brightness-only update (no class change, runs every hop)
- `T:<threshold>` — set confidence threshold on device
- `OFF` — extinguish LEDs

**Category mapping** lives in two places: `yamnet_serial.py` has the full `YAMNET_TO_CATEGORY` dict (521 YAMNet classes → 8 glasses categories), and `yamnet_mic_test.py` has a lighter keyword-based `NOISE_COLOR_MAP`. Keep these consistent when adding new sound categories.

**TFLite models** (`tflite_models/`, gitignored) are generated by `yamnet_esp32_feasibility.py`. The int8 model (~0.9 MB) is the ESP32 deployment target. The float32 model (~3.7 MB) is used for desktop accuracy comparison.

**Brightness** in `yamnet_serial.py` is RMS amplitude mapped through a dB range (`MIN_DB`/`MAX_DB`) with exponential smoothing (`SMOOTHING = 0.35`). Tune `MIN_DB`/`MAX_DB` for your environment if LEDs are always dim or always full.

## Key constants to know

| Constant | File | Default | Effect |
|---|---|---|---|
| `WINDOW_SEC` | mic_test, serial | 2.0 s | Audio window fed to YAMNet |
| `HOP_SEC` | mic_test, serial | 0.5 s | Inference refresh rate |
| `MIN_DB` / `MAX_DB` | serial | -50 / -10 | Brightness dB range |
| `SMOOTHING` | serial | 0.35 | LED brightness smoothing factor |
| `threshold` | serial | 0.50 | Min confidence to send a class command |

## Hardware target

- **Board**: XIAO ESP32-S3 (Seeed) with 8 MB PSRAM — required because YAMNet activations exceed the 320 KB internal RAM
- **Mic**: INMP441 I2S MEMS microphone
- **LEDs**: NeoPixel strip on GPIO 48 (configurable via `LED_PIN` in `.ino`)
- **Arduino IDE**: Board = ESP32S3 Dev Module, Flash = 8 MB, PSRAM = OPI PSRAM, USB = Hardware CDC and JTAG

## Dependencies

No `requirements.txt` exists yet. Install per-script:

```bash
pip install tensorflow tensorflow-hub sounddevice numpy scipy pyserial librosa soundfile soundata tqdm
```

The `soundata` package auto-downloads UrbanSound8K on first run of `yamnet_urbansound_test.py`.

## Known issues

- Full int8 YAMNet has a PAD kernel bug in the desktop TFLite runtime — `yamnet_accuracy_test.py` and `yamnet_urbansound_test.py` mark affected results as "PAD bug" and fall back to float32 where needed.
- `yamnet_esp32_feasibility.py` tries direct TFLite conversion first, then falls back to a SavedModel export path if the hub model doesn't expose concrete signatures.
